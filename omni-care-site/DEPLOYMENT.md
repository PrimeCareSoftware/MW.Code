# Omni Care Site - Deployment Guide

## Overview

The omni-care-site is a marketing/landing page for the Omni Care platform built with:
- Vite
- React
- TypeScript
- Tailwind CSS
- shadcn/ui

## Automated Build Process

A GitHub Actions workflow automatically builds the site whenever changes are pushed to:
- The `main` branch
- The `omni-care-site/` directory

The workflow is defined in `.github/workflows/deploy-omni-care-site.yml`.

### Build Artifacts

After each successful build, the compiled static files are saved as artifacts in GitHub Actions for 30 days. These artifacts are ready to be deployed to any static hosting provider.

## Deployment Options

### Option 1: Lovable Platform (Recommended for Lovable Projects)

Since this project was created with Lovable, you can deploy directly through their platform:

1. Open [Lovable](https://lovable.dev/projects/REPLACE_WITH_PROJECT_ID)
2. Click on "Share" â†’ "Publish"
3. Your site will be deployed automatically

To connect a custom domain:
1. Navigate to Project > Settings > Domains
2. Click "Connect Domain"
3. Follow the instructions to configure your DNS

Learn more: [Setting up a custom domain](https://docs.lovable.dev/features/custom-domain#custom-domain)

### Option 2: Manual Deployment from GitHub Actions Artifacts

1. Go to the [Actions tab](https://github.com/PrimeCareSoftware/MW.Code/actions)
2. Find the latest successful "Build Omni Care Site" workflow run
3. Download the `omni-care-site-build` artifact
4. Extract the contents
5. Upload to your hosting provider

### Option 3: Deploy to Netlify

[![Deploy to Netlify](https://www.netlify.com/img/deploy/button.svg)](https://app.netlify.com/start)

1. Connect your GitHub repository
2. Set build command: `cd omni-care-site && npm run build`
3. Set publish directory: `omni-care-site/dist`
4. Deploy!

### Option 4: Deploy to Vercel

1. Install Vercel CLI: `npm i -g vercel`
2. Navigate to the site directory: `cd omni-care-site`
3. Run: `vercel`
4. Follow the prompts

Or deploy via Vercel dashboard:
1. Import your GitHub repository
2. Set root directory: `omni-care-site`
3. Vercel will auto-detect the framework (Vite)
4. Deploy!

### Option 5: Deploy to Cloudflare Pages

1. Go to [Cloudflare Pages](https://pages.cloudflare.com/)
2. Connect your GitHub repository
3. Set build command: `npm run build`
4. Set build output directory: `dist`
5. Set root directory: `omni-care-site`
6. Deploy!

### Option 6: Custom Hosting (VPS/Hostinger/etc.)

1. Build the site locally or download from GitHub Actions artifacts
2. Upload the contents of the `dist` folder to your web server's public directory
3. Configure your web server (Apache/Nginx) to serve the static files

Example Nginx configuration:
```nginx
server {
    listen 80;
    server_name omnicare.com.br www.omnicare.com.br;
    root /var/www/omni-care-site;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## Local Development

To run the site locally:

```bash
cd omni-care-site
npm install
npm run dev
```

The site will be available at http://localhost:5173

## Building Locally

To build the site locally:

```bash
cd omni-care-site
npm run build
```

The build output will be in the `dist` directory.

## Troubleshooting

### Build fails with "MODULE_NOT_FOUND"
- Ensure all dependencies are installed: `npm ci`
- Check Node.js version: should be 20.x

### Changes not reflecting on the live site
- Ensure you're deploying the latest build from the main branch
- Clear your browser cache or try incognito mode
- Check if your hosting platform has automatic deployments enabled

### Styling issues
- Make sure Tailwind CSS is configured correctly
- Verify that all custom styles in `tailwind.config.ts` are properly set
